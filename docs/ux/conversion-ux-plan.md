# Sovereign Brief — 전환율 극대화 UX 개선 계획

> 작성일: 2026-05-14  
> 관점: UI/UX 디자이너 (전환율·참여도 중심)  
> 범위: 메인·아카이브·이메일·구독 전 페이지

---

## 진단 요약

현재 Sovereign Brief는 **정보를 나열**하고 있다.  
유저가 원하는 건 **"지금 내가 뭘 알아야 하는가"** 에 대한 답이다.

> *데이터 → 상태 → 결론 → 행동* 의 흐름이 끊겨 있다.

---

## 페이지별 문제 진단 & 개선 방향

---

### 1. 메인 페이지 (홈)

#### 1-1. 히어로 인라인 배지

**현재**
```
• INDEX  56  중립 구간  |  STATE  안정 상승  |  신호 보통
```

**문제**
- `56`이 무엇을 의미하는지 컨텍스트 없음. 좋은 건지 나쁜 건지 모른다.
- "중립 구간"은 흥미를 유발하지 않는다. 아무 긴장감도 없다.
- "신호 보통" — 가장 치명적. 첫인상에서 "별거 없음"을 전달한다.
- 구독 CTA (`무료 구독 →`)와 배지 사이에 연결고리가 없다.
  - 유저는 "56이 뭔지도 모르는데 왜 구독해?"라고 생각한다.

**개선 방향**
```
• INDEX  56 ↑4  |  BullQuiet  변동성 낮음  |  신호 강함
         어제 52        조용한 강세      추세 추종 유리
```

- `↑4` / `↓3` : 전일 대비 변화 추가 → 숫자가 살아있다는 느낌
- "신호 보통" → 구체적 의미로 교체: "추세 추종 유리" / "관망 권고" 등
- 배지 아래 한 줄 훅: **"오늘 진입 조건 확인 →"** (아카이브 링크)

**CTA 연결 흐름 개선**
```
[배지: 오늘 BTC 상황] → [훅 한 줄] → [구독 폼] → [무료 구독 →]
```
지금은 배지와 폼이 단절되어 있다. 훅 한 줄이 다리 역할을 해야 한다.

#### 1-2. SovereignLiveStrip (스크롤 후 첫 번째 섹션)

**현재**
```
SOVEREIGN INDEX         SOVEREIGN STATE
56 / 100                안정 상승
상승  중립 구간           변동성 낮음  신호 보통
[게이지 바]              BULLQUIET
```

**문제**
- 히어로 배지와 동일한 정보 반복. **더 크게 보여줄 뿐** 더 많은 걸 주지 않는다.
- "BULLQUIET"이 영어 기술 코드명이라 한국 유저에게 의미 전달 안 됨.
- 4개 정보(변동성, 신호, 국면, 코드)가 나열되어 있어 우선순위가 없다.
- 게이지 바가 있는데 "56이 어느 위치"인지 구간 표시가 부족하다.

**개선 방향**

좌측 (Index) — "지금 어디 있나"
```
56                어제보다 +4
중립 구간          ████████░░░░  [약세 | 중립 | 강세] 범례 포함
"30일 중 상위 40% 구간"         ← 상대적 위치
```

우측 (State) — "어떤 국면인가 + 그래서?"
```
조용한 강세           [BullQuiet]
변동성 낮음 · 완만한 상승 추세

→ 지금은 리스크가 가장 낮은 매수 구간입니다.
   오늘 브리프에서 구체적 진입 조건 확인 →
```

핵심: **상태 → 의미 → 행동**의 흐름. 마지막 줄이 CTA가 된다.

#### 1-3. "TODAY'S BRIEF" 샘플 섹션

**현재**
- 헤드라인 + 4개 마켓 심볼 + 뉴스 3개

**문제**
- 헤드라인이 왜 중요한지 설명이 없다.
- 마켓 심볼 숫자(BTC 가격 등)가 그냥 숫자다. 방향성/해석 없음.
- "오늘 브리프 먼저 읽기" CTA가 너무 작고 약하다.

**개선 방향**
- 헤드라인 위에 1줄 context: *"오늘의 핵심 판단:"*
- 마켓 심볼에 `+2.3% ↑` / `-1.1% ↓` 전일 대비 변화 추가
- 뉴스 3개 → 2개로 줄이고, 각각에 **왜 지금 중요한가** 한 줄 추가
- CTA를 섹션 중앙, 더 눈에 띄게

### 2. 아카이브 목록 페이지

**현재**
- 날짜 리스트. 각 항목: 날짜 + (브리프 제목 추정)

**문제 (코드 미확인 부분, 추정)**
- 날짜만 있으면 "클릭할 이유"가 없다.
- 유저가 "오늘이 특별한 날"인지 알 수 없다.

**개선 방향**
각 행에:
```
2026-05-12   BullQuiet · 56   "달러 약세 반전 신호 — 진입 기회 vs 함정"
2026-05-11   Choppy · 48      "혼재 신호 — 포지션 축소 권고"
```
- INDEX 점수 + 국면 → 한눈에 어떤 날인지
- 헤드라인 미리보기 → 클릭 욕구 유발

### 3. 아카이브 상세 페이지

#### 3-1. SovereignCommandPanel

**현재**
```
[SOVEREIGN INDEX]              [SOVEREIGN STATE]
BTC 시장 복합 지표               시장 국면 · 실시간 추적
56  상승  중립 구간              안정 상승
[게이지 + 범례 + 정확도 박스]    변동성 낮음  신호 보통
지수 구성 신호 (PC1 텍스트)      이유 태그  Gate 배지
```

**문제**
- 패널이 **분석 도구처럼** 생겼다. 브리프를 읽으러 온 유저에게 어렵다.
- "지수 구성 신호 PC1: fng(+0.48)..." — 기술 용어를 그대로 노출.
- "실전 검증 배지"는 신뢰를 높이지만, 위치와 크기가 묻혀 있다.
- Gate decision(vol_regime_v2)은 일반 독자에게 의미 없다.

**개선 방향**

상단 요약 바 (한 줄):
```
오늘 시장: 안정 상승 (BullQuiet) · 변동성 낮음 · 리스크 최소 구간
```

그 아래 두 패널 유지하되 언어 재작성:

좌측:
```
Sovereign Index    56 / 100
────────────────────────────
중립 구간 (40~60)
어제보다 +4p 상승 중

예측 정확도: 63.2%  (73회 검증)
```

우측:
```
오늘의 시장 국면
────────────────────────────
조용한 강세 (BullQuiet)

· 변동성이 낮고 완만한 상승 추세
· 공포탐욕 지수 정상 범위 유지
· 신호 신뢰도: 보통

→ 리스크를 최소화한 포지션이 유리한 구간
```

PC1 텍스트:
```
지수 구성: FNG · 뉴스 감성이 상승 주도, 롱숏비 부담 완화
```
→ 기술 코드 제거, 일반인이 읽을 수 있는 문장으로

#### 3-2. JudgmentBlock

**현재**
- 헤드라인만 있거나 + summaryLead + 상세 맥락

**문제**
- 헤드라인이 신문 헤드라인처럼 서술형일 경우, "그래서 나는?" 이 빠진다.

**개선 방향**
```
[오늘의 판단]
달러 약세 반전 신호가 확인됐습니다.

[핵심]
BTC는 단기 상승 모멘텀 유지 중이나, 달러 반등 시 조정 가능.

[지금 주목할 것]
→ 105K 저항선 돌파 여부 (오늘 22:00 기준)
```

헤드라인 → 핵심 → 오늘 볼 것. 3단 구조로 정리하면 유저가 "무엇을 얻었는지" 명확해진다.

### 4. 이메일 발송 본문

#### 4-1. 구독 확인 메일

**현재**
```
제목: [SOVEREIGN BRIEF] 구독 확인이 필요합니다
본문: 아래 버튼을 눌러 뉴스레터 구독을 완료해 주세요.
CTA: [구독 확인하기]
```

**문제**
- 제목이 **관료적**이다. 클릭율이 낮을 구조.
- 본문이 기능 설명만 한다. 유저가 왜 구독했는지 상기시키지 않는다.
- CTA 이후 무슨 일이 일어나는지 예고가 없다.

**개선 방향**

```
제목: 확인 한 번이면 오늘 BTC 브리프가 내일 아침 도착합니다
```

본문:
```
[SOVEREIGN BRIEF]

내일 아침, 시장이 움직이기 전에 알아야 할 것들을 정리해 드립니다.

아래 버튼을 누르면 구독이 완료됩니다.

[구독 완료하기 →]

완료하시면:
· 매일 오전 브리프 이메일 수신
· 오늘 브리프 아카이브 바로 접근 가능
· 언제든지 1클릭 구독 해지
```

---

#### 4-2. 브리프 발송 본문 (실제 뉴스레터)

*현재 템플릿이 `render-mail-shell.ts`에 있으나 브리프 본문 구성 코드 별도 확인 필요*

**공통 개선 방향**

**제목 공식 (매일 변경)**:
```
[수치 변화] [국면] + [오늘의 핵심 이슈] 
예: "BTC 56p ↑4 · BullQuiet — 달러 반전 신호 주목"
```
→ 제목에서 이미 오늘의 상황을 전달. 열지 않아도 가치 전달.

**본문 구조 (현재 unknown → 제안)**:
```
1. 오늘의 한 줄 판단 (맨 위)
2. INDEX/STATE (숫자 + 방향 + 의미)
3. 오늘의 핵심 이슈 2-3개 (짧게)
4. 마켓 스냅샷 (BTC, ETH, VIX, DXY)
5. 전체 보기 CTA → 아카이브 링크
```

**제목 open rate를 높이는 심리 트리거**:
- 긴박감: "오늘 주목할 단 하나의 신호"
- 수치 변화: "BTC 지수 +4p 급등 — 이유는?"
- 의외성: "신호는 강한데 왜 조심해야 하는가"

### 5. 구독 확인/해지 페이지

#### 5-1. 구독 확인 성공 페이지

**현재**
```
✓ 구독이 완료됐습니다.
[홈으로] [아카이브 보기]
```

**문제**
- 구독 완료 = 전환 성공 순간인데, 아무 감흥이 없다.
- "이제 뭘 하면 되는지" 안내가 약하다.

**개선 방향**
```
✓ 구독 완료!

내일 아침부터 브리프가 도착합니다.
오늘 브리프는 지금 바로 읽어보세요.

[오늘 브리프 읽기 →]          (강조 CTA)
[홈으로 돌아가기]              (보조)
```

오늘 날짜의 INDEX/STATE 요약 한 줄을 여기에도 표시하면 "이미 가치를 받고 있다"는 느낌을 준다.

#### 5-2. 구독 해지 페이지

**현재**
```
이메일: user@example.com
[취소] [Unsubscribe]
```

**문제**
- 해지를 너무 쉽게 만들었다. 리텐션 시도가 전혀 없다.
- 해지 후 "다시 구독" 경로가 없다.

**개선 방향**
```
정말 해지하시겠습니까?

지난 30일간 브리프 28회 수신했습니다.
최근 INDEX: 56 (안정 상승 구간)

[일시 중단하기 — 2주 후 재개]     ← 이탈 방지 옵션
[해지하기]                         ← 더 작게, 회색
```

해지 완료 후:
```
구독이 해지됐습니다.
마음이 바뀌시면 언제든 다시 시작하세요.

[다시 구독하기]
```

## 전체 페이지 공통 개선 원칙

### 원칙 1: "숫자에는 반드시 방향이 있어야 한다"

| 현재 | 개선 |
|------|------|
| `56` | `56 ↑4` |
| `안정 상승` | `안정 상승 (+4p, 3일째)` |
| `신호 보통` | `추세 추종 유리` |

### 원칙 2: "상태 → 의미 → 행동" 3단계 흐름

모든 지표 표시는 이 순서를 따른다:
1. **상태**: 지금 무엇인가 (BullQuiet, 56, 변동성 낮음)
2. **의미**: 그게 왜 중요한가 (리스크 최소 구간)
3. **행동**: 그래서 지금 뭘 봐야 하는가 (오늘 브리프 → 진입 조건)

### 원칙 3: "기술 용어는 두 번째 줄로"

```
[일반 독자 레이어]   조용한 강세 · 변동성 낮음
[기술 레이어]        BullQuiet · VIX 낮음 · RV 낮음 · FNG 정상
```

첫 줄은 누구나 읽는다. 기술 코드는 아래에 작게.

### 원칙 4: "CTA는 항상 오늘의 데이터에 연결"

```
❌ "오늘 브리프 읽기"
✅ "오늘 BullQuiet 구간 분석 전체 읽기 →"
```

제네릭한 버튼 텍스트보다 오늘 날짜/국면을 담은 텍스트가 클릭율 높다.

### 원칙 5: "비교 컨텍스트로 앵커링"

- `56 / 100` → `56 / 100 (30일 평균 51)`
- `신호 보통` → `신호 보통 (지난 7일: 2회 강함, 5회 보통)`
- `변동성 낮음` → `변동성 낮음 (VIX 18 — 90일 하위 25%)`

숫자는 비교 대상이 있을 때 의미가 생긴다.

## 작업 우선순위

### Phase 1 — 즉시 효과 (1~2일)
1. 히어로 배지: 전일 대비 변화 (`↑4`) 추가
2. 히어로 배지: "신호 보통" → 상황 기반 문구로 교체
3. SovereignLiveStrip 우측 패널: 국면 설명 한 줄 추가 (상태 → 의미)
4. 구독 확인 메일 제목 교체

### Phase 2 — 핵심 전환 개선 (3~5일)
5. SovereignCommandPanel PC1 텍스트 한국어 자연어로 재작성
6. SovereignLiveStrip "오늘 브리프 전체 읽기" → 국면 포함 CTA 텍스트
7. 아카이브 목록 행에 INDEX 점수 + 헤드라인 미리보기 추가
8. 브리프 이메일 제목 공식 수립 + 적용

### Phase 3 — 리텐션 강화 (1주)
9. 구독 확인 성공 페이지 재설계 (오늘 데이터 포함)
10. 구독 해지 페이지 리텐션 옵션 추가
11. JudgmentBlock 3단 구조 적용
12. 아카이브 상세 페이지 StatePanel "그래서?" 한 줄 추가

| 지표 | 현재 예상 | 개선 후 예상 |
|------|----------|------------|
| 히어로 → 구독 전환율 | 낮음 (배지-CTA 단절) | 배지 훅 추가 시 +20~40% |
