# Requirements — 금융 뉴스 데일리 프론트 페이지

> Version 2.0 | 2026-03-21
> 목적: 현재 파이프라인이 생성하는 미국 시장 브리핑을 한국어 정적 웹 페이지로 제공한다.

---

## 개요

이 프로젝트의 프론트엔드는 아침에 발행되는 미국 기술주·비트코인 브리핑을 **빠르게 읽고 판단할 수 있는 공개 웹 페이지**를 제공한다.

- 데이터 생산자: 현재 저장소의 파이프라인 서브시스템 (`src/morning_brief`)
- 데이터 소비자: 정적 프론트엔드
- 데이터 전달 방식: 날짜별 JSON + 최신 JSON
- 사용자 언어: 한국어 중심
- 제품 성격: 로그인 없는 공개 브리핑 페이지

프론트엔드는 **데이터를 다시 계산하거나 생성하지 않는 표시 레이어**다.  
판단, 브리핑, 뉴스 해석, 품질 상태는 파이프라인이 만든 결과를 읽고 구조화해서 보여준다.

## 기능 요구사항

### REQ-001 — 최신 브리핑 랜딩

- `/` 는 항상 최신 브리핑 1건을 보여준다.
- 로그인 없이 접근 가능해야 한다.
- 브리핑 생성 시각과 기준 시각을 상단에서 확인할 수 있어야 한다.
- 최신 데이터가 없으면 빈 화면 대신 상태 메시지를 표시해야 한다.

### REQ-002 — 상단 핵심 수치

- 상단에는 주요 시장 수치를 한눈에 읽을 수 있는 티커 또는 스트립 형태가 있어야 한다.
- 기본 우선순위는 다음을 따른다.
  - 미국 금리
  - DXY
  - VIX
  - 원/달러 환율
  - 나스닥 선물
  - 미국 대표 지수
  - BTC 현물
- 값이 없는 항목은 빈칸이나 `0` 대신 실패 상태를 명시해야 한다.
- 전일 캐시 대체값은 별도 표시가 있어야 한다.

### REQ-003 — 오늘의 판단과 브리핑 본문

- 페이지 중심에는 오늘의 판단과 브리핑 본문이 있어야 한다.
- 핵심 판단 문구는 브리핑 본문과 시각적으로 분리되어야 한다.
- 브리핑 본문은 마크다운을 유지한 채 렌더링해야 한다.
- `degraded` 또는 `critical` 상태에서는 신뢰도 경고가 함께 보여야 한다.

### REQ-004 — 토픽 요약

- 거시경제, 미국 주식, 빅테크/AI, 비트코인 토픽 요약을 카드 또는 블록으로 제공한다.
- 각 토픽에는 한 줄 요약, 핵심 수치 또는 연관 종목이 포함될 수 있다.
- 생성되지 않은 토픽은 빈 카드 없이 자연스럽게 생략한다.

### REQ-005 — 뉴스 피드

- 헤드라인 뉴스는 제목, 출처, 발행 시각, 한 줄 투자 맥락을 함께 보여준다.
- 사용자는 토픽별로 뉴스를 필터링할 수 있어야 한다.
- 1티어 출처는 시각적으로 구분되어야 한다.
- 클릭 시 원문으로 이동할 수 있어야 한다.
- 해석이 없는 뉴스도 목록에는 남길 수 있지만, 해석이 있는 뉴스보다 덜 강조되어야 한다.

### REQ-006 — X 시장 시그널

- X 시그널이 있는 날에는 별도 섹션으로 표시한다.
- 각 시그널에는 내용, 시장 영향, 센티먼트, 게시 시각이 포함되어야 한다.
- 데이터가 없으면 섹션 전체를 숨긴다.

### REQ-007 — 미국 증시와 기술주 보드

- 미국 대표 지수와 주요 기술주 흐름을 본문에서 확인할 수 있어야 한다.
- 주요 기술주는 등락률 절대값 기준으로 정렬할 수 있어야 한다.
- 현재가와 등락 방향이 함께 표시되어야 한다.
- 종목 데이터가 없으면 해당 행에 상태 표시를 제공한다.

### REQ-008 — 비트코인 전용 섹션

- BTC 섹션에는 최소 다음이 포함되어야 한다.
  - BTC 현물 가격
  - 공포탐욕지수 값과 레이블
  - 공식 ETF 총 BTC 보유량
  - 공식 ETF 총 AUM
  - 발행사별 공식 스냅샷
- 신뢰도가 낮은 추정치나 stale snapshot 기반 값은 사용자에게 노출하지 않는다.
- 공식 ETF 데이터가 없으면 보조 블록은 숨기고, 핵심 BTC 블록에는 상태 문구를 남긴다.

### REQ-009 — 아카이브

- `/archive` 에서 날짜 목록을 볼 수 있어야 한다.
- `/archive/[date]` 에서 해당 날짜 브리핑 상세를 볼 수 있어야 한다.
- 아카이브 상세 페이지는 메인과 같은 정보 구조를 유지한다.
- `/archive` 목록 페이지에는 TickerBar를 표시하지 않는다.

### REQ-010 — 부가 산출물

- 브리핑 본문은 마크다운 다운로드를 지원해야 한다.
- `llms.txt` 를 제공해야 한다.
- RSS 피드를 제공해야 한다.
- 페이지별 메타데이터와 OG 메타를 제공해야 한다.

### REQ-011 — 데이터 품질 상태

- 데이터 품질 상태는 `ok`, `degraded`, `critical` 중 하나로 표현한다.
- `ok` 는 경고 없이 표시한다.
- `degraded` 는 상단 경고 배너를 표시한다.
- `critical` 은 상단 경고와 함께 본문에서도 신뢰도 경고를 강조한다.
- 어떤 상태에서도 미완성 UI나 빈 화면을 노출하지 않는다.

### REQ-012 — 누락값, 캐시값, 섹션 노출 규칙

- 핵심 블록은 값이 없더라도 조용히 사라지지 않아야 한다.
  - 오늘의 판단
  - 핵심 수치
  - 브리핑 본문
  - 주요 뉴스
  - BTC 핵심 블록
- 보조 블록은 값이 없으면 숨길 수 있다.
  - X 시그널
  - 토픽 카드 일부
  - ETF 발행사 세부 현황
- 캐시 대체값은 사용자가 식별할 수 있어야 한다.
- null 데이터는 `N/A` 또는 상태 문구로 표시한다.

### REQ-013 — 업데이트 주기와 발행 기준

- 공개 브리핑은 **KST 08:00 기준 발행 경험**을 목표로 한다.
- 실제 페이지 갱신 시점은 파이프라인 완료 + 데이터 게시 완료 시점이다.
- 파이프라인 실패 시에는 전날 공개본을 유지해야 한다.
- 현재 페이지가 어떤 시점 기준인지 항상 보여야 한다.

### REQ-014 — 언어, 접근성, 반응형

- 서비스 언어는 한국어 중심으로 유지한다.
- 티커, 고유명사, 출처 브랜드명만 필요한 범위에서 영어를 사용한다.
- 모바일과 데스크탑 모두 사용 가능해야 한다.
- WCAG 2.2 AA 수준을 목표로 한다.
- 색상만으로 상태를 전달하지 않고 기호나 라벨을 함께 사용한다.

### REQ-015 — 성능과 배포

- 정적 생성 기반으로 빠르게 열려야 한다.
- 배포 대상은 Cloudflare Pages 다.
- 데이터 저장소는 Cloudflare R2 다.
- 프론트는 빌드 시점에 JSON을 읽어 정적으로 페이지를 만든다.
