# 수익률 개선 우선순위 — SJM 제외 전체 로직 분석 (2026-07-15)

> **범위**: SJM 섀도우 판단(M-3, 2026-08-10)을 제외한 현행 전체 로직에서 "높은 수익률"을
> 위해 남은 개선 항목. 이 문서 작성 과정에서 검증 대기 항목 2개(R4b·R6)를 실측해 종결시켰고,
> 그 결과를 반영해 우선순위를 재구성했다.
> 선행: [big-candle-no-pnl-diagnosis-20260715](big-candle-no-pnl-diagnosis-20260715.md),
> [remaining-improvements-20260710](remaining-improvements-20260710.md),
> [priority-improvements-20260714](priority-improvements-20260714.md).

## 헤드라인

**파라미터 튜닝으로 짜낼 수 있는 것은 사실상 소진됐다.** 게이트 완화·청산 시간배리어(Tier1)·
ATR 목표가(Tier2)·물타기 제거(R4b)·시즌럴리티(R6)·EWMA 사이징(R2)이 전부 백테스트/WF/DSR-PBO
게이트에서 기각 또는 무효과. 남은 것은 (1) **판단 인프라의 정확도**(비용 정합·MFE 정밀도·성과
회계), (2) **새 정보를 추가하는 피처**(FNG 지속기간·청산 데이터 대체소스), (3) **구조적
천장**(롱온리 하락장 한계)의 3계층이며, 이 순서가 곧 우선순위다. 정확하지 않은 판단 인프라
위에서 피처를 추가하면 또 잘못된 채택/기각을 반복한다.

## 1. 우선순위

### P0. 성과 회계 분리 — "지금 로직"의 트랙레코드부터 정확히 (공수 소)

**문제**: 누적 -15.4%(가중합, 24건)라는 숫자가 개선 판단을 오염시키고 있다. params_version별
분해(arena-status 섹션2)를 보면:

```
구버전(v14~v24, 16건): 합계 -14.1%   ← 누적 손실의 92%. WI-1/7·P-A 이전 로직
현행(v26~v30,   8건): 합계  -1.3%   ← 지금 실제로 돌아가는 로직의 성적
```

즉 "수익률이 왜 이러냐"의 대부분은 **이미 수정된 구버전 로직의 유산**이고, 현행 로직의
라이브 표본은 8건·거의 보합이라 아직 판정 불능 상태다. 구버전 거래를 섞어 보면 (a) 현행
로직을 과소평가하고 (b) 이미 고친 문제를 또 고치려는 시도(이번 Tier2가 부분적으로 그랬다
— MFE 진단 표본에 구버전 거래 포함)를 유발한다.

**액션**:
1. `arena_status.py` 섹션2·3(성과·MFE)에 `--since-version v26` 류 필터 또는 "현행 버전만"
   서브테이블 추가 — MFE 포착률도 현행 버전 거래만으로 재계산.
2. 대시보드(arena.sovereignwon.com)에 버전 구분 표시 검토(제품 투명성 관점 — "로직 개선
   이력이 보이는 트랙레코드"는 오히려 셀링포인트).

### P1. 백테스트 비용 정합 — 모든 A/B 판단의 기울기 제거 (공수 소) ⚡즉시

**문제**: 라이브 비용 모델은 왕복 ~13bps(fee 5/leg + slip 1/leg + spread 1 RT,
arena-cost-v2)인데, 분석 스크립트들이 쓰는 `BacktestSettings()` 기본값은
`slippage_bps=0.0`·`spread_bps_round_trip=0.0` → 왕복 10bps(`backtest.py:73-74`).
CLI 경로는 cost scenario를 주입하지만 **wi_tuning·exit_tuning·walk_forward_validate·
fng_optimize 등 채택 판단에 쓰인 하니스 전부가 기본값(10bps)으로 돌았다.**

**영향**: 왕복당 3bps 과소계상 — 모든 A/B가 회전율 높은 config에 체계적으로 유리.
특히 "거래 54→94(회전↑)"로 채택된 P-A atr1.0→v29, "거래 98→106"의 WI-7 같은
회전 증가형 채택이 재검증 대상. 3bps × 거래 40건 차이 = 기간 수익 1.2%p 왜곡 — 지금
알고들의 기간 수익이 ±수%인 걸 감안하면 결론이 뒤집힐 수 있는 크기.

**액션**:
1. `BacktestSettings` 기본값을 live cost scenario와 일치시키거나(slippage 1/spread 1),
   분석 하니스들이 명시적으로 cost scenario를 주입하도록 수정. (기본값 수정이 누락 재발
   방지에 유리 — 하니스가 늘어날 때마다 주입을 기억할 필요 없음.)
2. 최근 회전 증가형 채택 2건(P-A target·WI-7 target)을 13bps로 재실행해 결론 유지 확인.
   뒤집히면 해당 플래그 재조정 + PARAMS_VERSION bump.

### P2. MFE/MAE 1m 정밀화 (R8 승격) — 청산 개선 스레드의 진위 판정 (공수 소)

**문제**: "청산이 이익을 흘린다"(MFE 포착률 음수)는 진단이 Tier1·Tier2 두 캠페인을
발동시켰고 둘 다 기각됐다. 진단 자체가 4h봉 high/low 기반 보수 추정인데, 지금까지의
기각 패턴은 "진단이 가리키는 방향으로 메커니즘을 넣어도 개선이 안 됨" — **진단의 해상도
문제일 가능성**을 의심할 차례다. 4h high는 포지션이 실제로 잡을 수 있었던 이익이 아닐 수
있다(스파이크 1분 뒤 반락 등).

**액션**: `arena_realtime_feature_bars`(1분, 2.5만행+)로 라이브 기간 거래의 MFE/MAE를
1m 정밀도로 재계산(P0의 현행 버전 필터와 결합). 결과에 따라:
- 1m 포착률이 실제로 양호 → **청산 개선 스레드 전체 종결** (더 이상 exit 튜닝 안 함),
  이후 개선은 진입 품질·피처로 집중.
- 1m에서도 명확히 음수 → "붙잡을 수 있었던 이익"이 실존 — 그때 1m 근거로 메커니즘 재설계.

### P3. R5: FNG 지속기간 피처 (`fng_days_below_30`) — 남은 유일한 진입 품질 레버 (공수 소~중)

fng_contrarian은 현행 버전 기준으로도 라이브 최대 관찰 대상이고(진입 빈도 최다),
공포 1일차(뉴스 쇼크, 계속 하락 가능)와 공포 N일 지속(매도 소진)은 평균회귀 품질이
다르다는 근거가 확립돼 있다. FNG 히스토리는 parquet에 전량 보유 — 수집 작업 없이
파생 컬럼 + regimeRaw 노출 + wi_tuning A/B로 즉시 검증 가능. **P1(비용 정합) 완료 후**
그 하니스로 검증할 것(순서 중요 — 지금 하니스로 하면 또 기울어진 판단).

검증 설계: `fng_days_below_30 >= N`(N∈{2,3,5}) 진입 게이트 vs 사이징 가중(1일차 ×0.5 등)
두 변형 → WF 6윈도 → DSR/PBO. 게이트형은 거래수 급감 시 검증 불능이 되므로(WI-4 전례)
사이징형이 살아남을 가능성이 더 높음.

### P4. unknown 레짐 진입 차단 A/B — 조이기는 미검증 가설 (공수 소, 기대 중)

라이브 손실의 레짐 분포: fng 손실 진입 8건 중 unknown 5, vix_rsi 5건 중 unknown 4.
백테스트 M-2에서도 macd 손실 전원이 unknown 진입. 금지된 것은 게이트 **완화**이고,
**조이기**(unknown 시 진입 보류 또는 사이징 ×0.5)는 아직 A/B된 적 없다.

⚠️ 주의 2가지: (1) unknown이 전체 봉의 40%라 게이트형은 거래 반토막 — 사이징형 우선.
(2) vix_rsi는 구조 게이트 추가 시 항상 악화됐던 전례(WI-5 기각) — fng·vix_rsi 각각
독립 A/B로, 하나라도 기각되면 그 알고는 접을 것. SJM이 unknown 자체를 줄이는 상위 해법
이지만(제외 대상), 이 A/B 결과는 SJM 승격 판단 때 "unknown 회피의 가치" 정량 근거로도
재사용된다.

### P5. WI-9 청산 데이터 대체 소스 — 새 정보를 추가하는 유일한 경로 (공수 중, 결정 필요)

P3·P4까지는 기존 정보의 재가공이다. 강제청산 데이터(매도 소진 확인)는 fng·omnibus
역발산 진입의 품질을 원리적으로 개선할 수 있는 **신규 정보원**인데, fstream 지역차단으로
수집이 0건(진단 완료, [priority-improvements](priority-improvements-20260714.md) P1-3).

**액션**: REST 서드파티(CoinGlass / Coinalyze / coinank 등) 비교 — 무료 티어 유무·4h 집계
가능 여부·과거 데이터 백필 가능 여부(백필이 되면 30일 대기 없이 즉시 백테스트 검증 가능
— 스트림 방식 대비 큰 이점). **제공자·비용은 사용자 결정 사항.** 결정되면 수집기 교체는
공수 중(기존 `arena_liquidation_bars` 스키마 재사용).

### P6. 구조적 천장 — 롱온리 현물의 하락장 한계 (전략 결정, 장기)

현행 6알고 전부 현물 long/flat. 90일 낙폭 -21%·MA200 하회 시장에서 이 구조의 이론적
최선은 "현금 보유 ≈ 0%"다 — 즉 **지금 국면에서 '높은 수익률'은 어떤 파라미터로도 불가능
하고, 할 수 있는 최선은 손실 회피**(실제로 regime_trend·macd가 그렇게 하는 중). 하락에서
+를 만들려면:
- (a) 선물 숏 트랙 추가 — 스팟 아레나 정체성·리스크 정책과 충돌, 제품 결정 필요.
  omnibus DOWN_TREND조차 반등 롱만 매핑되는 것은 의도된 설계(`spot_short_no_trade`).
- (b) 스테이블 파킹 수익(현금 슬리브에 무위험 수익 부여) — 페이퍼트레이딩에선 회계만
  추가하면 되고 현실성 있음(실거래 전환 시 실제 가능한 구조).
- (c) 현상 유지 — "하락장은 쉬는 것도 성과"를 대시보드 서사로 명시(노출률 함께 표기).

(b)가 공수 대비 효과가 가장 현실적. (a)는 사용자 결정 없이는 진행 불가.

### P7. 프로세스 방어 — 수익률을 "지키는" 항목 (공수 소)

이번 점검 주기에서 실제 손실을 낸 것은 알고가 아니라 인프라였다(v30 라벨 버그로 전후비교
오염, WI-9 59시간 무음 사망). 재발 방지 2건, 둘 다 기존 백로그에 있으나 미착수:
1. **PARAMS_VERSION CI 체크**: 커밋 메시지에 `arena-params-v\d+`가 있는데 `parameters.py`
   상수와 불일치하면 실패 (P0-1 재발 이미 2회: v25, v29/v30).
2. **주간 /arena-status 루틴화 + row-count 헬스체크**: "로그상 정상"이 아니라 실산출물
   (신규 거래 수·버킷 카운트·버전 라벨 일치) 기반 이상 감지.

### P8. 장기 (착수 조건 미충족)

- **메타라벨링(R9-P4)**: triple-barrier 라벨 데이터셋 파이프라인은 지금 구축 가능하나,
  모델 학습은 라이브 표본 확대 후. P0~P2로 판단 인프라 정비가 선행돼야 라벨 품질이 나옴.
- **MAB 메타 포트폴리오 섀도우(R9-P8)**: 7번째 가상 트랙. 알고별 기대값이 안정적으로
  분화된 후에나 의미 — 우선순위 최하 유지.

## 2. 실행 순서 요약

```
✅ 완료(2026-07-21) P1 비용 정합 → P0 성과 회계 분리 → P2 1m MFE 정밀화
                    → P3 FNG 지속기간 A/B(기각) → P4 unknown 조이기 A/B(기각)
결정 대기          P5 청산 API 제공자(사용자) · P6-(a) 숏 트랙(사용자)
병행(공수 소)      P6-(b) 스테이블 파킹 회계 검토 · P7 CI 체크·루틴화
종결(재시도 금지)   §0 전 항목 + R4b·R6·P3·P4 (실측 완료)
```

P1~P4는 전부 실행·문서화 완료([arena-status-review-20260721](arena-status-review-20260721.md) §8~§10).
파라미터 튜닝으로 짜낼 수 있는 것은 이제 확정적으로 소진 — 남은 항목은 전부 사용자 결정
(P5/P6-a) 또는 저효과 유지보수(P6-b/P7)뿐이다.

**요지**: 순서가 내용만큼 중요하다. P1을 건너뛰고 P3/P4를 돌리면 기울어진 하니스로 또
채택/기각을 쌓는다. P0~P2는 전부 공수 소·시장 무관·즉시 실행 가능이고, 이 셋이 끝나면
"현행 로직이 실제로 얼마나 벌고 있는가"를 처음으로 정확하게 알게 된다 — 그 위에서 P3
이후의 피처 작업이 의미를 가진다.
