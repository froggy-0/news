## P-A. fng_contrarian 이익 포착 메커니즘 (최우선 실험)

### 근거 데이터 (교차 검증)
1. **라이브 MFE (2026-07-09 실측)**: 청산 6건 평균 MFE **+2.09%** / 평균 MAE -2.28% /
   평균 실현 **-1.41%**. 손실 5건 중 4건이 보유 중 한때 **+1% 이상** → 이익이 존재했다가
   증발. v22 체계(물타기+시간손절, 가격손절 없음) 하의 v24 진입 4건도 flat 청산 손실 3건
   (-1.53/-0.39/-1.17%, 승리 +1.01% 1건).
2. **백테스트 v22 진단(기존)**: time_stop 만기보유 승률 86% vs 조기 flat 36% — 보유 중
   이익 회복이 실재함을 독립 확인.
3. **WI-2(청산 히스테리시스=보유 연장) 백테스트 기각(전 변형 Δ-0.71~-1.03)**:
   "더 오래 보유"는 답이 아님. 라이브 MFE와 종합하면 데이터가 가리키는 방향은
   **"이익이 있을 때 잡기"(profit capture)** — 보유 연장이 아니라 익절 메커니즘.
4. **이론 정합**: 평균회귀 수익은 초기 봉에 집중(Kaminski·Lo) → profit target이 정석.
   MFE/MAE 문헌: 포착률<30%면 청산 최적화가 진입 최적화보다 우선
   (traderssecondbrain.com/guides/mae-mfe-analysis).
5. **선례**: omnibus v28 `target_exit` — 동일 메커니즘으로 백테스트 -6.24→-4.57(Δ+1.67)·
   승률 57→61% 검증 완료. **인프라(execution_rules.target_exit_triggered, stream 감시,
   backtest 봉high 체결) 전부 재사용 가능**.

### 실험 설계 (wi_tuning 그리드)
| 변형 | 내용 | 비고 |
|---|---|---|
| A | baseline (현행 v28) | |
| B | 목표가 = 평단 + ATR×{1.0, 1.5, 2.0} | omnibus REBOUND와 동일 패턴 |
| C | 목표가 = bb_mid 복귀 | 평균회귀 이론 정합(밴드 중앙) |
| D | MFE 트리거 트레일: +1% 도달 후 트레일링 활성화 | 가격손절 금지 원칙(Kaminski·Lo)을 **이익 구간 한정**으로 보존 — 손실 방향 스톱은 여전히 없음, 이익 보호만 |
| E | B×히스테리시스 결합 (목표가 있는 상태에서 FNG<45 보유) | WI-2 기각이 "목표가 부재" 조건이었는지 확인 |

- **물타기 상호작용 필수 확인**: 목표가 기준은 트랜치 체결 후 **가중평균 평단**. 평단이
  내려가면 목표가도 재계산(진입 시 고정이 아니라 scale-in 시 갱신 — omnibus와 다른 점).
- 파라미터: `FNG_TARGET_EXIT_ENABLED`(플래그, 기본 off) + `FNG_TARGET_MODE` +
  `FNG_TARGET_ATR_MULT`. close_reason `target_exit` 재사용.
- 검증: 6개월 macro 백필 백테스트(fng n=54 — 표본 충분) + DSR/PBO + time_stop 경로
  무회귀 + 타 알고 무영향 확인.

### 채택 기준
fng sum_w 개선 **+ 승률 개선 또는 유지** + plateau 존재(단일 뾰족값 기각) + DSR 참고.
통과 시 `arena-params-v29` 후보.

## P-B. vix_rsi — 괴리 원인 규명 완료, 모니터링 전환 (코드 변경 없음)

### 근거 데이터
- 라이브 청산 5건 전수 조사(2026-07-10): **전부 v18~v24 진입** —
  6/22(+0.65 v18), 6/23(-0.37 v20), 6/24(-3.18 v20), 6/30(-2.52 v24), 7/8(-0.15 v24).
- v26(진입 안정화 + 청산 히스테리시스, 7/8 저녁 EC2 반영)이 정확히 이 실패 모드를
  타겟했음 — 코드 주석의 "라이브 id16 -2.52%"가 위 6/30 거래.
- fresh 백테스트 +3.94%(6알고 중 최고)는 **v26+v28 기본값 기준** → 괴리의 주원인은
  실행 문제가 아니라 **파라미터 세대 차이**.

### 판정·액션
- WI-5(MA200 게이트/크로스 트리거) 재실험 불요 — 백테스트에서 이미 baseline 최고 확인.
- **액션**: v26 이후 진입 표본 n≥10 축적까지 관찰. `/arena-status`의 params_version 분리가
  자동 추적. n≥10 시점에 v26+ 표본만으로 라이브 PF 재평가 — 그때도 PF<1이면 실행 슬리피지/
  체결 타이밍(4h 봉종가 vs :05 체결) 조사로 승격.

## P-C. 주간 백테스트 잡 수리 (운영, 소형)

### 근거 데이터
- 저장 백테스트 최신 = 2026-06-20, **params v7**, 이후 20일 무갱신. scheduler에
  `_run_weekly_backtest_safe` 잡은 등록돼 있음(로그 확인) — 실행되는데 저장이 실패하거나,
  실행 자체가 실패 중.
- 영향: `/arena-status` 섹션5(라이브 vs 백테스트)의 기준선이 죽어 있어 매번 ⚠️STALE.

### 액션
1. EC2 로그에서 주간 잡 실행 흔적·에러 확인 (`journalctl -u arena | grep -i backtest`).
2. 원인 수정 후, 주간 잡이 **macro 백필 없이** 돌고 있다면(arena_macro_snapshots만 사용)
   현행 그대로 두되 결과 해석에 "라이브 macro 커버 구간만 유효" 명시. macro 백필 통합은
   parquet 의존성(로컬 파일)이라 EC2 잡에는 넣지 않음 — 필요 시 로컬 `--fresh-backtest`.

## P-D. WI-9 청산 스트림 수집 시작 (즉시 실행 가능, 비용 최소)

### 근거
- 인프라 배포 완료(v27) + 테이블 생성 완료. 그러나 `ARENA_LIQUIDATION_STREAM_ENABLED`
  env가 off → **축적 0일째**. 30일 시계는 켜는 날부터 시작.
- fng(P-A)·omnibus REBOUND의 "매도 소진" 확인 피처 후보 — 데이터가 없으면 영원히 후보.

### 액션
EC2 systemd 환경변수에 `ARENA_LIQUIDATION_STREAM_ENABLED=true` 추가 + 재시작 →
`SELECT count(*), max(bar_start) FROM arena_liquidation_bars`로 24h 내 수집 확인.
(선물 스트림 별도 커넥션 — 트레이딩 무영향 설계 확인 완료.)
