## Phase 0 — 기반 구축 (완료)

**기간:** 2026년 초 ~ 2026년 6월
**목표:** 데이터 수집 + 신호 체계 + 백테스트 결과 확보

### 달성 항목

| 항목 | 내용 | 상태 |
|---|---|---|
| 거시 데이터 파이프라인 | VIX (FRED), FNG (CoinGecko), Regime State | 완료 |
| 기술 지표 계산 | RSI(14), MACD(12/26/9), BB(20) — 순수 Python | 완료 |
| 백테스트 기반 신호 검증 | BullQuiet 116건 / 61.2% 적중 | 완료 |
| 신호 트랙레코드 시스템 | Supabase signal_log, 148건 실시간 기록 중 | 완료 |
| 아레나 코어 구현 | 5개 알고리즘, 4H 사이클, 실시간 스톱로스 | 완료 |
| EC2 배포 준비 | systemd, deploy 스크립트 | 완료 |

**Phase 0 핵심 발견:**
- BullQuiet Regime에서만 신호를 내는 것이 올바른 전략
- BearPanic에서의 숏 포지션은 코인플립 수준 (50%) → 분리 처리
- LLM 브리핑 비용 대비 효과 없음 → 비활성화, 알고리즘 검증에 집중

## Phase 1 — 포워드 테스트 누적 (현재)

**기간:** 2026년 7월 ~ 2027년 1월 (목표 6개월, 이상적으로 12개월)
**목표:** 통계적으로 유의미한 실시간 트랙레코드 1,000건 이상 확보

### 이 단계에서 무엇을 하는가

1. **아레나 서버 상시 가동** (EC2 Seoul)
   - 5개 알고리즘이 4시간마다 독립적으로 신호 발생
   - Binance 실시간 가격으로 스톱로스 실행
   - 모든 진입·청산이 Supabase에 타임스탬프와 함께 영구 기록

2. **데이터는 쌓이지만 아무에게도 보여주지 않는다**
   - 공개 시점을 강제로 미루는 이유: 초기 데이터는 노이즈가 많음
   - 6개월 이상 지나야 "이 알고리즘은 이런 장에서 작동한다"가 보임

3. **알고리즘 성과 내부 모니터링**
   - 주별로 5개 알고리즘 승률·수익률 내부 리뷰
   - 명백히 broken인 알고리즘 발견 시 → 수정 후 버전 업 (기존 기록 유지)

### Phase 1 성공 기준

| 기준 | 목표값 |
|---|---|
| 포지션 기록 수 | 1,000건 이상 |
| 상위 알고리즘 승률 | 55% 이상 (통계적 유의성 확보) |
| 시스템 가동률 | 99% 이상 |
| 최악 알고리즘 처리 | 명확한 underperformer 1~2개 교체 또는 유지 결정 |

### 주요 마일스톤

```

2026-08  500건 돌파, 중간 점검

## Phase 2 — 공개 리더보드 런칭

**기간:** 2027년 1분기 (조건: Phase 1 성공 기준 충족 시)

### 무엇을 공개하는가

1. **웹 리더보드 (기존 Next.js 프론트엔드 확장)**

```

2. **공개 원칙 (차별화 포인트)**

## Phase 4 — 플랫폼 확장 (2028 이후)

**목표:** 알고리즘 마켓플레이스 + 실거래 브리지 (선택적)

### 알고리즘 마켓플레이스

- 외부 개발자가 자신의 알고리즘을 아레나에 등록
- 공개 트랙레코드로 검증 → 구독자 확보 시 수익 쉐어
- 플랫폼은 중립적 평가 인프라 역할

### 실거래 브리지 (가장 신중하게 접근)

```
신뢰받은 알고리즘 (18개월 이상, 승률 60%+)
    ↓
Upbit / Binance 자동매매 연동 옵션 제공
    ↓
사용자가 자신의 API 키로 직접 연결 (플랫폼은 중개 안 함)
```

- **주의:** 실거래 연동은 금융업 규제 리스크 최고 구간
- 법률 자문 완료 후 결정
- 구조: "신호를 따라 사용자 자신이 거래" 방식으로 책임 분리

## 알고리즘 발전 로드맵

### 현재 운영 중인 5개

```
regime_v2       — BullQuiet=long, 기타=flat          (거시 기반)
fng_contrarian  — FNG<30=long, FNG>70=short          (역발산)
vix_rsi         — VIX 안정 + RSI<50 = long           (복합)
macd_momentum   — MACD histogram이 ATR 대비 유의미할 때 방향 추종 (기술)
multi_factor    — BullQuiet 롱 / BearPanic 숏 + RSI/MACD 확인      (복합)
```

### 추가 예정 (데이터 확보 후)

| 알고리즘 | 추가 조건 | 예상 추가 시기 |
|---|---|---|
| exchange_outflow | 거래소 순유출 급증 = long | CryptoQuant API 확보 후 |
| onchain_accumulation | 장기 보유자 축적 신호 = long | Glassnode 데이터 확보 후 |
