## 1. 현황 진단

### 1-1. 신호 발화 조건 (현재)

```
VIX < 90일 rolling q40
AND BTC realized_vol_20d < 45일 rolling q45
AND FNG < 70
```

### 1-2. Hit vs Miss 특징 비교

신호가 발화했지만 7일 수익이 음수였던 날(miss, n=29)과 양수였던 날(hit, n=54)의 차이:

| 특징 | hit 평균 | miss 평균 | p값 | 해석 |
|------|---------|---------|-----|------|
| `btc_return_7d` (현재 7d 수익) | +0.6% | +2.1% | **0.029** | miss 날은 이미 오른 뒤 |
| `btc_realized_vol_20d_lag1` | 0.016 | 0.018 | **0.025** | miss 날은 RV가 약간 더 높음 |
| `fng_value` | 39.7 | 40.8 | 0.453 | 차이 없음 |

**핵심 발견**: miss의 원인은 **VIX/FNG가 아니라 직전 모멘텀 과열**이다.  
신호가 이미 오른 구간에서 발화할 때 실패율이 높다.

## 2. 내부 다듬기 (신규 피처 불필요)

### 2-1. 모멘텀 역배열 필터 ← 가장 큰 개선 여지

저변동성 구간에서 **직전 7일이 하락했던 날**은 평균 반등 확률이 현저히 높다.

| 조건 | n | hit rate | mean_ret | coverage |
|------|---|---------|---------|---------|
| 기본 (vol+FNG) | 76 | 0.711 | 2.18% | 14.3% |
| + ret7_lag < 0 (직전 하락) | 29 | **0.862** | **3.28%** | 5.5% |
| + ret7_lag < -3% (강한 하락) | 13 | **0.923** | **4.74%** | 2.5% |
| + ret7_lag > 0 (직전 상승) | 47 | 0.617 | 1.50% | 8.9% |

**메커니즘**: 저변동성 구간에서 단기 가격 하락은 정보 없는 소음 매도로, 평균 회귀 압력이 작동한다. 반면 직전 상승 후 저변동성은 "오른 뒤 쉬는 구간"으로 추가 상승 동력이 약하다.

**트레이드오프**: coverage가 14.3% → 5.5%로 감소. 약 60일에 1번 발화 수준.

### 2-2. Taker 불균형 필터 ← 두 번째 개선 여지

매수 체결 압력(taker buy zscore)이 낮을 때 신호 품질이 높다.

| 조건 | n | hit rate | mean_ret | coverage |
|------|---|---------|---------|---------|
| taker_z > 0.5 (매수 쏠림) | 28 | 0.571 | 1.14% | 5.3% |
| taker_z -0.5~0.5 (중립) | 24 | 0.792 | 2.22% | 4.5% |
| taker_z < -0.5 (매도 쏠림) | 24 | **0.792** | **3.35%** | 4.5% |

**메커니즘**: 저변동성 + 매도 체결 우세 = 스마트머니 조용히 축적 중. 반대로 저변동성인데 매수 쏠림이 강하면 이미 포지션이 쌓인 상태 → 추가 상승 여력 제한.

**권고**: `taker_imbalance_zscore_30d_lag1 ≤ 0.5` 조건 추가.  
coverage: 14.3% → 9.0%, hit: 0.711 → 0.792.

## 3. 신규 피처 분석

### 3-1. 전체 피처 fwd_ret_7d 상관관계 (Spearman, 전체 기간)

유의미(p<0.05)한 피처 13개 / 전체 69개:

| 피처 | r | p | 방향 해석 |
|------|---|---|---------|
| `etf_total_aum_usd` | **-0.246** | 8e-9 | AUM 높으면 선반영 완료, 상승 여력 줄어듦 |
| `etf_total_btc` | -0.202 | 2e-6 | 위와 동일 |
| `open_interest_usd` | -0.200 | 3e-6 | OI 높으면 롱 레버리지 과잉, 청산 위험 |
| `btc_ma_200d` | -0.189 | 1e-5 | MA 절댓값 높음 = 상승 구간 후반부 |
| `vix_lag1` | +0.121 | 0.005 | VIX 약간 높을 때 오히려 반등 여지 |
| `oi_price_divergence_score_7d` | -0.122 | 0.005 | OI↑ 가격↑ 정배열 = 레버리지 과잉 경고 |
| `funding_rate` | -0.090 | 0.037 | 펀딩 높으면 롱 과잉 |

**현재 미수집 피처 중 가장 가치 있는 것들:**

#### A. 거래소 순유출 (Exchange Net Outflow) — 최우선 구현 대상
- BTC가 거래소를 떠난다 = 장기 보유 의도 = 실제 수요
- OI, ETF AUM의 "간접 신호"를 직접 측정하는 가장 강력한 대체재
- 현재 `sources/exchange_outflow.py` 스캐폴딩 완료, API 연결만 남음
- **예상 효과**: taker imbalance보다 노이즈 적은 누적 수요 측정

#### B. 매크로 맥락 (DXY, 10Y Yield) — 중기 추가
- 현재 VIX만으로는 주식시장 공포를 감지하나 달러 강세/약세 맥락 없음
- DXY 상승(달러 강세) 구간에서 BTC 반등 신호 신뢰도 낮아짐
- FRED API 이미 연결되어 있어 추가 수집 비용 낮음

#### C. BTC Options 데이터 (Put/Call ratio, 25d skew) — 장기 추가
- 옵션 시장이 spot/futures보다 선행하는 경향 있음
- Deribit API 필요, 수집 난이도 높음

#### D. 검색 트렌드 / 소셜 볼륨 — 낮은 우선순위
- Google Trends "bitcoin" 급증 = 리테일 고점 신호
- FNG와 상관관계 높아 중복 정보 가능성

## 5. 결론: 신규 피처 vs 내부 다듬기

| 구분 | 효과 | 난이도 | 권고 |
|------|------|-------|-----|
| 내부 다듬기 (taker 필터) | hit +8%p, 구현 1일 | 낮음 | **즉시** |
| ETF inflow 방향 활용 | 소폭 개선 예상 | 낮음 | **즉시** |
| Exchange outflow (신규) | 독립 정보량 최대 | 중간 | **다음 스프린트** |
| Macro 피처 (DXY) | 국면 의존성 완화 | 낮음 | **다음 스프린트** |
| Options 데이터 | 선행 지표 | 높음 | 나중에 |

**한 줄 결론**: 내부 다듬기(taker 필터)로 즉시 hit을 높일 수 있고, 신규 피처(exchange outflow)가 장기적으로 신호 국면 의존성을 낮추는 근본 해결책이다. 둘 다 해야 한다.
