## 1. 출발점 — 연구 질문

이 프로젝트는 단 하나의 질문에서 시작됐다.

> **"뉴스 감성이 실제 BTC 가격 등락을 선행 설명하는가?"**

직관적으로 그럴듯한 가설이다. 긍정적인 뉴스가 많이 나오면 투자자들이 낙관적으로 반응해 매수 → 가격 상승. 하지만 이 인과 방향이 실제로 맞는지, 데이터가 확인해 줄 것인지는 별개다.

이 질문에 답하려면 세 가지가 필요했다:
1. 뉴스의 "감성"을 수치로 만드는 방법 (비정형 → 정형 변환)
2. 뉴스 외에 BTC 가격에 영향을 주는 다른 시장 구조 지표들
3. 어느 쪽이 어느 쪽을 앞서는지 통계적으로 검증하는 방법

## 2. 데이터 수집 — 무엇을 왜 모았나

### 2.1 비정형 데이터: 뉴스

| 소스 | 내용 | 수집 방법 |
|---|---|---|
| 암호화폐 뉴스 | 매일 발행되는 BTC/시장 관련 기사 | briefs/{YYYY-MM-DD}.json 백필 |

핵심 품질 규칙: `sentimentStatus == "skipped"` 또는 기사 수 ≤ 1인 날짜는 분석에서 제외. 기사 1개로는 대표성 없는 감성이 통계를 오염시키기 때문이다.

### 2.2 정형 데이터: 시장 지표

아래 7가지가 선택된 이유는 각각 서로 다른 시장 참여자 심리·행동을 측정하기 때문이다.

| 피처 | 측정 대상 | 선택 이유 |
|---|---|---|
| `fng_value` | Fear & Greed Index (0~100) | 소매 투자자의 종합 심리 — 시장 방향과 강한 상관 |
| `vix` | S&P500 옵션 내재변동성 | 전통 금융 위험 회피 심리 → 고 VIX 구간에서 BTC 동반 하락 패턴 |
| `funding_rate` | 선물 미결제 자금 조달비율 | 레버리지 방향 — 양수면 롱 과열, 음수면 숏 과열 |
| `oi_change_pct` | 미결제약정(Open Interest) 변화 | 시장 참여 강도 — 증가 = 새 포지션, 감소 = 청산 |
| `etf_net_inflow_usd` | 비트코인 현물 ETF 순유입 | 기관 자금 흐름 (2024년 ETF 승인 이후 핵심 지표) |
| `usdkrw_return` | 달러-원 환율 변동 | 한국 시장 리스크온/오프 프록시, 아시아 자금 흐름 |
| `btc_log_return` | BTC 일봉 로그 수익률 | 가격 자체의 모멘텀 (Granger 검정의 역방향 확인용) |

**왜 VIX인가**: 암호화폐는 전통 자산과 디커플링된다는 주장이 있지만, 실제로 2022~2024년 데이터에서 VIX 스파이크 구간에 BTC가 동반 하락하는 패턴이 반복됐다. "위험회피" 심리가 전 자산군에 걸쳐 작동하기 때문이다.

**왜 USDKRW인가**: 원화 약세는 한국 투자자들의 달러 자산 선호 증가를 시사하고, 반대로 원화 강세는 위험자산 매수 여력을 뜻한다. 아시아 시간대 BTC 거래량이 크기 때문에 의미 있는 변수다.

## 3. 비정형 데이터 처리 — FinBERT로 감성 수치화

### 3.1 모델 선택

**FinBERT** (`ProsusAI/finbert`): 금융 뉴스 텍스트로 파인튜닝된 BERT 기반 분류기. Positive / Negative / Neutral 3클래스 확률을 출력한다.

대안인 일반 BERT나 감성사전 기반 방법 대신 FinBERT를 택한 이유:
- 금융 도메인 특화 어휘 처리 (일반 모델은 "bearish", "sell-off" 같은 표현을 잘못 분류)
- 문맥 의존 분류 — "not bullish"를 부정 처리

**현재 한계**: FinBERT는 일반 금융 뉴스로 학습됐지 crypto 특화가 아니다. "HODL", "DeFi", "liquidation cascade" 같은 crypto 특유 표현을 일반 금융 문맥으로 해석한다. CryptoBERT로 교체하면 더 좋을 수 있지만 전체 감성 데이터 재생성 비용이 크다.

### 3.2 감성 점수 계산

```
뉴스 기사 텍스트
     ↓  FinBERT 추론
     │  P(positive), P(negative), P(neutral)  ← 3클래스 확률 합 = 1
     ↓
article_score = P(positive) − P(negative)  ∈ [−1, +1]
     │  P(neutral)은 공식에 포함되지 않음 → 중립 기사는 0에 가까운 값
     ↓
daily_score = 해당일 기사 article_score 평균
     (count ≤ 1인 날 제외)
```

`article_score`가 +1이면 완전 긍정, -1이면 완전 부정, 0에 가까우면 중립. P(neutral)을 공식에서 빼는 이유는 neutral 비율이 높은 기사가 점수를 왜곡 없이 0 근방에 머물도록 하기 위해서다. 하루치 기사들의 평균을 내면 당일 뉴스 전체의 감성 기류가 된다.

## 4. 정형 데이터 처리 — 결합·정렬·결측치 처리

### 4.1 UTC 기준 Inner Join

모든 데이터를 **UTC 날짜**를 기준으로 결합한다. Binance 선물 데이터는 UTC 00:00 기준, 뉴스 감성도 UTC 날짜 기준으로 집계된다. 타임존 불일치로 인한 날짜 오정렬을 막기 위해 변환 시 UTC를 강제한다.

결합 방식: Inner Join → 모든 소스에서 데이터가 있는 날짜만 유효 행으로 사용.

### 4.2 Forward Fill (결측치 보간)

주말·공휴일에 VIX, USDKRW 등은 값이 없다. 이를 처리하는 방식:

| 변수 | 최대 ffill 일수 | 이유 |
|---|---|---|
| BTC 가격 | 2일 | 암호화폐는 24/7 거래 — 이틀 이상 공백은 데이터 오류 |
| USDKRW | 3일 | 주말+월요일 공휴일까지 커버 |
| VIX | 2일 | 주말만 커버, 그 이상은 시장 구조 변화 가능성 |

Forward Fill의 철학: "오늘 값이 없으면 어제 값이 여전히 유효하다"는 가정. 최대 일수를 제한하는 이유는 장기간 stale 데이터가 모델에 스며드는 것을 막기 위해서다.

### 4.3 Lag-1 정렬

예측 신호로 쓸 피처는 반드시 **하루 뒤로 밀어야(Lag-1)** Lookahead가 없다.

```
오늘(t) 뉴스 감성 → 내일(t+1) BTC 방향을 예측
→ 피처 컬럼명에 "_lag1" 접미사

오늘(t) FNG → 내일(t+1) 예측용으로 사용
→ fng_value_lag1
```

Lag-1을 적용하지 않으면 "오늘 BTC가 올랐으니 오늘 감성이 좋다"는 동시적 상관관계가 예측력처럼 보인다. 이것이 T+0 분석의 구조적 함정이다.

**T+7 타겟과의 관계**: 이 프로젝트의 예측 구조는 다음과 같다.

```
오늘(t) 보유한 피처값 (_lag1 포함)
     → t+7의 BTC 방향 예측
```

즉 피처가 `_lag1`으로 정렬되어 있어도, 예측 시점(오늘 t)에 해당 값은 이미 알려진 정보다. T+7 레이블(`btc_fwd_ret_7d`)은 오늘부터 7일 후 수익률이다. 이 7일 간격이 Embargo를 7일로 설정하는 이유이기도 하다.

## 5. 이상치 처리 — 왜 컬럼별 정책을 쓰는가

### 5.1 초기 접근의 문제점

> **IQR(사분위 범위)**: Q3 − Q1, 즉 중간 50% 데이터의 폭. `IQR×3` 마스킹은 중앙에서 IQR의 3배 이상 벗어난 값을 이상치로 판정한다. 평균·표준편차 기반 3σ보다 이상치에 덜 민감해 금융 데이터에 자주 쓰인다.

초기에는 **Row-level IQR×3 마스킹**을 사용했다: 어떤 컬럼이든 IQR×3을 넘으면 그 **행 전체**를 NaN 처리.

결과: 약 20.56%의 행이 제거됐다. 그런데 제거된 날들을 확인해 보니 2022년 Luna 사태, 2024년 ETF 승인, 시장 급등락일이 포함됐다. 이 날들은 "잘못된 데이터"가 아니라 **가장 중요한 시장 신호**였다.

### 5.2 컬럼별 분리 정책 (현재)

| 컬럼 | 정책 | 이유 |
|---|---|---|
| `usdkrw_return` | IQR×3 마스킹 | 공휴일 재개장 갭은 실제 데이터 오류에 가까움 |
| `volume_change_pct`, `oi_change_pct` | Winsorize (q01/q99) | 방향 정보는 보존, 극단적 크기만 압축 *(Winsorize: 분포 상·하 1%를 해당 분위값으로 클리핑)* |
| `btc_return`, `etf_net_inflow_usd`, `funding_rate` | 마스킹 없음 | 극단값 자체가 신호 — 급락, ETF 대규모 유입, 극단 펀딩비 |

**핵심 원칙**: 금융 시계열에서 "이상치"는 노이즈가 아닐 수 있다. 어떤 컬럼의 극단값이 데이터 오류인지 시장 신호인지를 컬럼별로 판단해야 한다.

## 6. PCA 합성 — 8개 피처를 하나의 점수로

### 6.1 왜 단일 점수가 필요한가

7개 시장 지표 + 뉴스 감성 = 8개 피처를 개별적으로 다루면 해석이 복잡하고, 피처 간 공선성(VIX↑ ↔ FNG↓ 처럼 강한 상관)이 모델을 불안정하게 만든다.

"시장이 지금 어떤 상태인가"를 하나의 숫자로 표현할 수 있다면 이후 분석이 훨씬 간결해진다.

### 6.2 왜 PCA인가

**포기한 대안들**:
- *단순 가중평균*: 어떤 근거로 가중치를 정할지 자의적. 공선성 처리 불가.
- *지도학습(Logistic, GBM)*: n=539에서 과적합 위험. 피처 가중치가 매일 바뀌어 운영 신뢰성 낮음.

**PCA 선택 이유**: 데이터가 스스로 알려주는 최대 분산 방향을 추출한다. 레이블(가격 방향)을 보지 않고 피처들만의 공분산 구조에서 "시장 상태"를 표현하는 가장 중요한 축을 찾는 비지도 방법이다.

**중요한 한계**: PCA 지수는 "지금 시장이 어떤 상태인가(Regime Descriptor)"를 잘 표현하지만, "내일 오를 것인가"를 직접 예측하지 않는다. 지수가 높다 = 지금 분위기가 낙관적이다 ≠ 추가 상승 예측. 실제로 PCA 지수를 그대로 신호로 쓴 결과 Hit Rate 47.6%로 오히려 감성 단독(48.9%)보다 낮았다. 이 실패 분석의 자세한 내용은 [signal-design-log.md §2-A](signal-design-log.md#2-a-pca-하이브리드-지수-자체를-신호로-쓰기)를 참조.

### 6.3 PCA 결과

```
8개 피처 입력:
  news_sentiment_mean_lag1, fng_value_lag1, vix_lag1,
  funding_rate_lag1, oi_change_pct_lag1, etf_net_inflow_usd_lag1,
  usdkrw_return_lag1, btc_log_return_lag1

VIF gate: 분산팽창계수(VIF) > 10.0인 피처 자동 제거 후 PCA
  ─ VIF(Variance Inflation Factor): 한 피처가 다른 피처들의 선형조합으로
    얼마나 설명되는지를 측정. VIF > 10이면 강한 다중공선성 존재.
  ─ 예: fng_value와 news_sentiment는 서로 높은 상관이지만 loading이
    비슷하게 분산돼 VIF 임계를 넘지 않아 둘 다 포함된다.

결과:
  PC1 설명 분산: 85.4%
  PC1 loading 구조:
    FNG↑   +0.53  (탐욕 = PC1 증가)
    감성↑  +0.53  (긍정 뉴스 = PC1 증가)
    VIX↓   −0.48  (변동성 감소 = PC1 증가)
    펀딩↑  +0.31  (롱 과열 = PC1 증가)
```

PC1이 85.4%의 분산을 설명한다 = 8개 피처에서 가장 중요한 공통 요소가 하나 있다는 뜻. 그것이 **"시장 위험선호 강도"**다.

### 6.4 부호 앵커와 최종 점수

PCA는 부호 방향이 자의적이다 (PC1이 "위험선호 = 높음"일 수도 있고 "위험회피 = 높음"일 수도 있다). 이를 고정하기 위해 **fng_value_lag1**과의 상관 방향으로 PC1 부호를 앵커링한다.

```
if corr(PC1, fng_value_lag1) < 0:
    PC1 = −PC1  ← 부호 반전 → "높을수록 낙관적" 방향으로 통일

최종 점수 = MinMax 정규화 → [0, 100]
  0  = 극단 공포/비관
  100 = 극단 낙관/위험선호
```

## 7. 통계 검정 — 인과성 탐색

### 7.1 정상성 검정 (ADF + KPSS)

Granger 검정은 시계열이 정상적(Stationary)일 때 유효하다. 먼저:
- **ADF 검정**: "단위근이 있다(비정상)" 귀무가설 검정
- **KPSS 검정**: "정상적이다" 귀무가설 검정

두 검정을 같이 쓰는 이유: ADF만 통과해도 KPSS가 비정상을 검출할 수 있다. 두 검정이 일치할 때만 정상성으로 판단한다.

### 7.2 Granger 인과성 검정

> **Granger 인과성**: "A의 과거값을 추가했을 때 B의 미래 예측력이 향상되면, A가 B를 Granger-cause한다"  
> 진정한 인과관계가 아니라 **예측력 선행성** 검정이다.

F-통계량: 값이 클수록 A가 B를 더 강하게 앞선다.

**검정 규모**:
- 순방향 검정: 뉴스감성→BTC, FNG→BTC, 펀딩비→BTC 등 18쌍
- 역방향 검정: BTC→뉴스감성, BTC→FNG, BTC→LSR 등 5쌍
- 각 lag = 1, 2, 3 → **전체 69개 검정**

69개를 같은 α로 검정하면 우연히 유의한 결과가 나올 확률이 급증한다 (Multiple Testing Problem). 이를 제어하기 위해 **BH-FDR(Benjamini-Hochberg)** 보정을 적용한다.

**왜 BH-FDR인가 (Bonferroni 대신)**: Bonferroni는 α/69 ≈ 0.00072를 threshold로 쓴다. 이렇게 엄격하면 진짜 신호도 통과하지 못한다(검정력 ≈ 0%). BH는 "거짓 양성 비율을 q% 이하로 통제"하는 방식이라 threshold가 완화된다. 단, 현재 표본(539행)에서 2%p 차이를 감지하려면 ~15,000행이 필요하다 — 파워가 8~10%에 불과하다.

## 8. 핵심 발견 — 방향이 뒤집혔다

Granger 검정 결과 (n=539, BH-FDR 보정 후):

```
방향                                    F-통계   해석
──────────────────────────────────────────────────────────
btc_log_return → long_short_ratio       F=483    ← 가격이 포지션을 즉각 바꿈
btc_log_return → fng_value              F=420    ← 가격이 공포탐욕지수를 만듦
btc_log_return → news_sentiment_mean    F=104    ← 가격이 뉴스 논조를 만듦
────────────────────────────────────────────────────────────
news_sentiment_mean → btc_log_return    F=  8    ← 뉴스→BTC: 52배 약함
```

**BTC가 오른 날 이후 며칠간 긍정적인 뉴스가 만들어진다.**

처음 질문("뉴스 감성이 BTC를 예측하는가")에 대한 답: **약하다**. 오히려 반대 방향이 훨씬 강하다.

이 발견이 프로젝트 방향을 바꿨다:

> "감성을 직접 예측 신호로 쓰는 게 아니라,  
> 시장 전체 구조에서 **저위험 구간을 골라내는 필터**를 만들자."

뉴스 단독 Hit Rate = 48.9% (동전 던지기보다 낮음)  
PCA 하이브리드 지수 단독 Hit Rate = 47.6% (더 낮음)

이것이 "PCA 지수 → 직접 예측"이 아니라 "변동성 필터(vol_regime_v2) + PCA 지수 조합"으로 가게 된 이유다.

> PCA 직접 예측 실패의 상세 분석(진입 횟수, 누적 수익, FP/FN 패턴): [signal-design-log.md §2-A](signal-design-log.md#2-a-pca-하이브리드-지수-자체를-신호로-쓰기)

## 9. Walk-Forward → 61.8%

### 9.1 vol_regime_v2 — 저위험 구간 필터

Granger 발견 후 방향이 바뀌었다. 감성/PCA 지수를 직접 쓰는 대신, "시장이 조용한 구간에서만 진입"하는 필터를 찾기로 했다.

**vol_regime_v2 조건**:
```
VIX < 90일 rolling q40    (단기 변동성 낮음)
AND
실현변동성(RV) < 45일 rolling q45   (BTC 자체 변동성 낮음)
AND
20 < F&G < 80              (극단 공포/탐욕 회피)
```

세 조건 모두 만족하면 진입 (coverage ≈ 56.2%).

**파라미터 근거**: VIX window/quantile, RV window/quantile은 2D grid search로 최적화됐다. F&G 20-80은 고정값 — rolling quantile을 시도했지만 시장 급변 때 역방향으로 움직여 절대값이 더 안정적이었다.

### 9.2 Walk-Forward Validation

"vol_regime_v2가 61.8%를 낸다"고 말하려면 이 수치가 **학습에 사용되지 않은 데이터**에서 나왔음을 보장해야 한다.

이 프로젝트에는 두 가지 WFV 경로가 있다:
- **경로 A**: 규칙 기반 신호(vol_regime_v2 등) 검증 — `train=120d / test=30d / ~13 fold`
- **경로 B**: ML 복합 점수(L1Logistic, ElasticNet, LightGBM 앙상블) 검증 — `train=240d / test=45d / 9 fold`

아래는 vol_regime_v2를 검증하는 경로 A 기준이다. 경로 B가 더 긴 train 기간을 쓰는 이유는 ML 모델이 피처 가중치를 안정적으로 추정하려면 더 많은 표본이 필요하기 때문이다. 두 경로의 상세 파라미터 비교는 [wfv-deep-dive.md §8](wfv-deep-dive.md#8-이-프로젝트의-실제-적용)을 참조.

경로 A (`walk_forward_validate`):

```
┌──────────────────────┐  [7일 Embargo]  ┌──────────┐
│   TRAIN (120일)      │ ──────────────► │ TEST 30일│  fold 1
└──────────────────────┘                 └──────────┘
         ↓ 30일 이동
  ┌──────────────────────┐  [7일 Embargo]  ┌──────────┐
  │   TRAIN (120일)      │ ──────────────► │ TEST 30일│  fold 2
  └──────────────────────┘                 └──────────┘
            ↓ 반복 (~13회)
```

539행 기준 `(539 − 157) / 30 ≈ 13 fold`

**Embargo 7일의 역할**: T+7 예측에서 Train 마지막 날의 정답(7일 후 가격)이 Test 초반과 겹치는 Label Leakage를 막는다.

**95% CI 계산**: Circular Block Bootstrap (n=1,000, block=14일). T+7 레이블이 7일씩 겹쳐 자기상관이 있으므로 일반 정규분포 CI가 너무 좁다. 14일 연속 블록으로 샘플링해 자기상관 구조를 보존한다.

### 9.3 결과

| 신호 | Hit Rate | 95% CI | 유의 |
|---|---|---|---|
| 뉴스 감성 단독 | 48.9% | [45.2%, 52.6%] | ❌ |
| PCA 하이브리드 지수 | 47.6% | [42.9%, 52.0%] | ❌ |
| **vol_regime_v2 (변동성 필터)** | **61.8%** | **[52.0%, 71.6%]** | **✅ CI 하한 > 50%** |

**왜 61.8%가 의미 있는가**: CI 하한이 52.0%로 50%를 초과한다. 운 좋은 구간에서 나온 수치가 아니라 13개 독립 fold에 걸쳐 Lookahead 없이 검증된 결과다.

**중요한 단서 — 두 층위의 판단**:

이 프로젝트는 신호 상태를 두 기준으로 동시에 평가한다:

| 기준 | 값 | 의미 |
|---|---|---|
| `decision = "promote"` | 롤링 overlay gate 통과 | 최근 21일 hit_rate ≥ 55%, coverage 정상, p-value 중앙 < 0.10 |
| `decision_strict = "research_only"` | BH-FDR 통계 미달 | fdr_q=1.0 — 539행으로는 파워 8~10%라 어떤 신호도 통과 불가 |

두 기준이 상충하는 이유: `decision`은 실용적 기준(최근 성과 유지), `decision_strict`은 통계적 엄밀성(69개 검정 BH 보정 후 유의) 기준이다. 현재는 통계 파워 부족으로 두 기준이 모순적으로 보이지만, 이는 데이터 부족의 구조적 한계다. 상세 설명: [wfv-deep-dive.md §9](wfv-deep-dive.md#9-검증이-보장하는-것--보장하지-않는-것)

- Worst Fold (Fold 1): Hit Rate 0.508 — 데이터 초기 구간 불안정성
- 운영 적용: 60일 drift 누적 확인 후 판단 예정

### 10.2 매일 생성되는 오늘의 점수

WFV 검증과 별개로, 매일 자동으로 오늘의 하이브리드 지수 점수가 생성된다.

```
전체 과거 데이터 (2024-11-30 ~ 어제)
     ↓  PCA fit + Scaler fit (이 데이터로만 학습)
오늘 데이터 (오늘 하루 — 학습에 쓰인 적 없음)
     ↓  transform-only
오늘의 하이브리드 지수 점수 → latest.json → 이메일·대시보드
```

이 방식을 **Expanding Window OOS**라고 한다. 데이터가 쌓일수록 학습 데이터가 늘어나고, 오늘 데이터는 항상 학습에 쓰인 적 없는 OOS 상태다.

예외: OOS 계산이 실패하면 in-sample 점수로 대체 (`today_score_method = "in_sample_fallback"`). 이 경우 해당 날짜 점수는 OOS 보장 없음 — artifact의 `todayScoreMethod` 필드로 구분 가능.

### 10.3 신호 트랙레코드 시스템

매일 오늘의 신호(vol_regime_v2 상태, PCA 점수 등)가 Supabase에 기록된다. 7일 경과 후 `fill-signal-outcomes`가 실제 BTC 수익률과 비교해 `hit` 여부를 업데이트한다. 90일 후부터 유의미한 트랙레코드 통계가 축적된다.
